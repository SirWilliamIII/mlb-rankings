# Project Workflow

## Development Cycle
1. **Plan:** Review the `plan.md` for the current track.
2. **Test:** Write a failing test for the next task.
3. **Implement:** Write code to pass the test.
4. **Refactor:** Clean up code while keeping tests green.
5. **Commit:** `git commit` with a clear message.

## Standards
- **Testing:** >80% coverage required.
- **Commits:** Conventional Commits format.
